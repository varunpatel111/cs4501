LeaseMe

By Arun Kanumuru, Ben Charles, and Varun Patel

ABOUT

Welcome to our project! Our idea is to create a marketplace to allow students to sublet their apartments to other students who are staying in Charlottesville over the summer. As of right now, we have two models: CustomUser and Listing. Below we have documented our models and how to call the CRUD methods.


MODELS

CustomUser

CustomUser has 4 fields: "first_name", "last_name", "email", and "password". email is an EmailField, and must be a valid email address. All other fields are strings. Passwords are not hashed as of now.

{"email": "email@gmail.com", "first_name": "John", "id": 1, "last_name": "Appleseed", "password": "Password123" }

Listing

Listing has 9 fields: "address", "num_bedrooms", "num_bathrooms", "price", "start_date", "end_date", "description", "sold", and "user". num_bedrooms, num_bathrooms, and price are integers. start_date and end_date are DateFields, and must be passed in as YYYY-MM-DD. sold is a boolean, and user is the primary key of the associated user. All other fields are strings.

{"address": "1819 JPA", "description": "Cool place!", "end_date": { "day": 13, "month": 12, "year": 2010 }, "id": 1, "num_bathrooms": 3, "num_bedrooms": 5, "price": 1232, "sold": false, "start_date": { "day": 12, "month": 12, "year": 2010 }, "user_id": 1}


CREATE

To create a new user, submit a POST request to http://localhost:8001/api/users/create/ with all 4 fields.
To create a new user, submit a POST request to http://localhost:8001/api/listings/create/ with all 9 fields.

READ

To see all existing users, submit a GET request to http://localhost:8001/api/users/.
To see all existing listings, submit a GET request to http://localhost:8001/api/listings/.

To read an existing user, submit a GET request to http://localhost:8001/api/users/{pk}/, where "pk" is the primary key of the user you want to get (without the brackets).
To read an existing listing, submit a GET request to http://localhost:8001/api/listings/{pk}/, where "pk" is the primary key of the listing you want to get (without the brackets).

UPDATE

To update an existing user, submit a POST request to http://localhost:8001/api/users/{pk}/ with all 4 fields, where "pk" is the primary key of the user you want to update (without the brackets).
To update an existing listing, submit a POST request to http://localhost:8001/api/listings/{pk}/ with all 9 fields, where "pk" is the primary key of the user you want to update (without the brackets).

DELETE

To delete an existing user, submit a DELETE method to http://localhost:8001/api/users/{pk}/, where "pk" is the primary key of the user you want to delete (without the brackets).
To delete an existing listing, submit a DELETE method to http://localhost:8001/api/listings/{pk}/, where "pk" is the primary key of the listing you want to delete (without the brackets).
